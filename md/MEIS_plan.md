# 最小嵌入推理系统 (Minimal Embedding Inference System, MEIS)
## 完整实施计划

---

## 0. 核心理念与系统定位

### 0.1 原始洞察

人类（以及任何"强智能体"）在面对未知时，做的其实是同一件事：**将新主张嵌入到已有的知识网络中，评价它与网络接壤处的"扰动程度"**。扰动越小（越能在不破坏已有信念的前提下被吸收）、跨域锚点越多（身高→密度→体重→鞋码→压强→脚印深度这类链条），该主张就越可能为真。

这一思路对应哲学史上三条线的合流：
- **Quine 的信念之网（web of belief）**：信念按中心-外围结构组织，修正优先发生在外围
- **Lakatos 的研究纲领**：硬核受保护带保护，挑战先击中保护带
- **贝叶斯更新的 KL 惩罚项**：后验分布在先验约束下对证据做最小修正

### 0.2 系统目标

MEIS 不是一个"回答问题"的系统，而是一个 **元评价系统**：

> 输入：一个新主张（自然语言） + 一组已有证据
> 输出：该主张在已知世界模型下的融贯度评分 + 支持它的跨域推理链 + 最具信息量的下一步观测建议

这和现有的 RAG、Agentic Science、POPPER 都不同：
- RAG 是前馈检索+生成，没有信念网络结构
- Agentic Science（如 The AI Scientist）在领域内做参数搜索，不做跨域结构识别
- POPPER 做假设的统计检验，不做信念网络扰动度测量

### 0.3 关键边界条件

**这不是 AGI 方案**。MEIS 只处理"可以被部分形式化的主张"——即主张中至少有一部分变量可以落位到结构化的概念空间（身高、体重、压强、价格、频率…）。完全不可形式化的主张（诗歌是否优美）不在范围内。

## 2. 分阶段实施路线图

### Phase 0 — 基础设施 (预计 2-4 周)

**目标**：搭建可运行的最小工程环境

**交付物**：
- Python + PyMC/NumPyro 环境 (L1)
- geomstats 或 Pyro 的信息几何模块 (L2)
- 暂时跳过范畴论实现，用 Python 类模拟 Markov 核的组合 (L3 占位符)
- OpenAI/Anthropic API 接入 (L4)
- 一个"定律动物园"数据集（20-30 个可解析的物理/化学/生物简单系统）

**关键决策**：
- 概率编程框架选型：推荐 **NumPyro**（JAX 后端，支持自动微分，便于后续 Fisher 信息计算）
- 如果要做结构层的严肃实现，考虑 **Gen.jl**（Julia，MIT ProbComp 组原生工具）

---

### Phase 1 — MVP：单域信念网络 + LLM 先验注入 (4-6 周)

**场景**：复现并扩展你提出的"Alice vs Charlie 体重"例子

**工作流**：

1. **用户输入**（自然语言）：
   > "Alice 比 Bob 高，Bob 和 Charlie 一样高。Alice 和 Charlie 谁体重重？"

2. **L4 调度**：LLM 解析出变量（Alice_height, Bob_height, Charlie_height, Alice_weight, Charlie_weight）与约束

3. **L4→L1 先验注入**：LLM 被提示"给出这些变量相关的人体常识先验"，例如：
   - 成人身高 ~ Normal(170, 10) cm
   - 成人密度 ~ Normal(1010, 30) kg/m³（几乎恒定）
   - 体型相似的成年人：weight ≈ density × k × height³（立方律加噪声）

4. **L1 执行信念传播**：NumPyro 定义生成模型，MCMC 采样，得出 P(Alice_weight > Charlie_weight) ≈ 0.85

5. **L4 返回**：输出后验概率 + 解释链（"由于身高正相关且人体密度接近恒定…"）

**难点与缓解**：
- **先验可信度问题**：LLM 给出的数值先验可能是幻觉。缓解：让 LLM 同时给出先验来源（论文/常识/估计），并让系统在 MCMC 中对先验本身做敏感性分析
- **变量识别歧义**：LLM 可能漏变量或建错结构。缓解：让 LLM 输出结构化的 JSON（变量名、类型、单位、关系），人工审校前几个案例

**成功标准**：
- 对 10 个类似"Alice-Charlie"风格的问题给出后验概率，5 个答案与人类共识一致
- 所有中间变量（身高、密度、体重）的后验分布可视化输出

---

### Phase 2 — 跨域先验传递 (6-8 周)

**场景**：引入脚印、鞋码等次级证据，验证后验收紧机制

**扩展工作流**：

1. 在 Phase 1 问题上新增证据：
   > "此外，已知 Alice 脚印在沙地里比 Charlie 的深；两人鞋码相同。"

2. **L1 扩展网络**，引入新节点：
   - foot_area ← 从 shoe_size 推断（带噪）
   - pressure = weight / foot_area
   - footprint_depth = g(pressure, soil_hardness)

3. **L4 注入跨域先验**：
   - 鞋码→脚面积的映射（常识表）
   - 压强→沙地深度的物理近似（含土壤硬度作为隐变量）

4. **L1 后验传播**：加入脚印观测后，P(Alice_weight > Charlie_weight) 应收紧到 ≈ 0.97

5. **L2 介入**：计算加入脚印观测前后的 Fisher 信息矩阵，量化"脚印深度"这一证据贡献了多少信息量

**关键工程挑战——跨域先验库的冷启动**：

这是整个项目最硬的骨头。两条可选路径：

| 策略 | 描述 | 优点 | 缺点 |
|---|---|---|---|
| A. LLM 运行时生成 | 每次新问题由 LLM 临时生成跨域关系 | 无需预建库 | 一致性差，同一关系每次数值可能不同 |
| B. 离线构建 + 运行时检索 | 预先让 LLM 枚举常见跨域关系，人工审校入库，运行时 RAG 检索 | 一致、可复用 | 覆盖率有限，需持续扩充 |

**建议采用混合策略**：核心物理/生理常识走 B（离线库），长尾关系走 A（LLM 即时）。这对应 Tenenbaum 所说的"intuitive physics engine"——核心是固定的，外围是动态的。

**成功标准**：
- 在"Alice 体重"案例上，随着证据增加，后验熵单调下降
- Fisher 信息矩阵能正确排序"哪个下一步观测最值钱"
- 跨域先验库初始规模 ≥ 200 条关系

### Phase 3 — 最小嵌入评价函数 (8-10 周)

**这是项目的理论核心**。前两阶段解决了"如何做一次推理"，本阶段解决"如何评价一个主张与已有信念网络的嵌入质量"。

**形式化**：

给定：
- 已有信念网络 $\mathcal{B}$ (N 个节点，M 条边，每条边带条件分布)
- 新主张 $h$（翻译为对某节点 $v_h$ 的新约束或新节点）

定义**扰动度**：

$$
D(h, \mathcal{B}) = \underbrace{D_{KL}(P(\mathcal{B}) \| P(\mathcal{B} | h))}_{\text{全局后验漂移}} + \lambda \cdot \underbrace{|\Delta \text{structure}|}_{\text{结构性修改数}}
$$

其中 $|\Delta \text{structure}|$ 统计为吸收 $h$ 需要新增/删除/修改的边数。

**操作化步骤**：

1. 用户给出候选主张集合 $\{h_1, h_2, \ldots, h_k\}$（例如：能剧男性垄断的多种解释）
2. 系统对每个 $h_i$ 计算 $D(h_i, \mathcal{B})$
3. 返回按扰动度升序排列的候选列表 + 每个候选修改了哪些边

**难点——结构性修改的计数规范**：

不同的图编辑距离定义会给出不同排序。建议采用 **BIC 分数差**：

$$
|\Delta \text{structure}| \propto \text{BIC}(\mathcal{B}) - \text{BIC}(\mathcal{B} + h)
$$

这样"引入新节点/边需要有数据支撑"被自动惩罚，避免过度拟合。

**文献关联**：
- 此处直接对接 Perrone (2024) 在 Markov 范畴态射集上定义散度的工作
- 与自由能原理 (Friston) 数学上同构：复杂度项 - 准确度项

**成功标准**：
- 在"能剧性别垄断"、"东汉早夭"等多解释案例上，系统输出的排序与领域专家直觉一致率 ≥ 60%
- 对明显不融贯的主张（如"Alice 比 Charlie 重因为她属虎"）给出显著更高的扰动度

### Phase 4 — 结构同构识别 (10-14 周)

**场景**：跨域迁移。系统学会过"身高→密度→体重→压强→脚印"的推理结构后，应能识别"电压→电阻→电流→功率→发热"是同构结构，直接迁移。

**技术路径**：

1. 每个信念网络子图表示为 Markov 范畴中的一个态射（或 string diagram）
2. 用图神经网络（GNN）或 graph-kernel 方法学习子图的结构嵌入
3. 新问题到来时，检索最近邻结构，作为"归纳偏置"初始化其信念网络

**这是最理论化的一层，也是范畴论+信息几何真正发挥作用的地方**。之前几轮讨论得出的结论：范畴论不是用来做一次推理，而是用来识别"两个不同领域的推理是否是同一抽象模式"。

**文献直接对接**：
- Fritz et al. (2020) Markov 范畴的 Blackwell–Sherman–Stein 定理给出了判定两个统计实验"等价"的范畴理论标准
- Lê et al. (2025, arXiv:2505.03862) 提供了 SPD 矩阵流形上概率态射的度量工具

**此阶段可选推迟**：
如果 Phase 1-3 已产出足够论文价值，Phase 4 可作为后续工作。对 PhD 论文规模而言，Phase 1-3 已构成完整贡献。

---

### Phase 5 — 闭环验证环境 (贯穿全程)

**借鉴 POPPER 但改造**：POPPER 对自然语言假设做统计检验，我们要做的是对已形式化假设做**融贯度评分**，所以需要自建测试床。

**定律动物园 v1** (Phase 0 产出的扩展)：

| 等价类 | 成员 (同构系统) |
|---|---|
| 指数衰减 | RC 电路、放射性衰变、一阶反应、遗忘曲线 |
| 阻尼振荡 | LC 电路、单摆、捕食者-被捕食者 |
| 饱和增长 | 逻辑斯谛、Michaelis-Menten、电容器充电 |
| 分配问题 | 重力场、电场、Nash 均衡 |

**评价任务**：

1. **假设提取**：给定一组时间序列，系统能否推出转移矩阵？
2. **等价类识别**：给定多个系统的数据，系统能否正确聚类？
3. **迁移预测**：对新系统 $S$，若系统识别 $S$ 属于已知等价类，能否用等价类模型预测 $S$ 的未见行为？
4. **最小扰动评分**：给定主张 "$S$ 属于等价类 $C_1$" 和 "$S$ 属于等价类 $C_2$"，扰动度排序是否正确？

**量化指标**：
- 任务 1：Frobenius 范数误差 < 0.1
- 任务 2：ARI (Adjusted Rand Index) > 0.8
- 任务 3：held-out MSE 相对基线降低 ≥ 30%
- 任务 4：专家一致率 ≥ 70%

## 6. 为什么这个路径是可行的

回顾之前的所有讨论，三个结论收敛到同一方向：

1. **原始直觉（最小嵌入推理）是对的，只是之前找错了锤子**。Quine、Lakatos、贝叶斯 KL 惩罚、自由能原理，都在说同一件事。
2. **范畴论+信息几何是度量工具，不是表示工具**。真正承担表示的是贝叶斯网络/概率程序。这个分工在 Phase 1-3 已经充分利用，Phase 4 才动用范畴论的真正威力（跨域同构识别）。
3. **真正的瓶颈是跨域先验库**，不是数学形式化。这正是 LLM 能贡献最大价值的地方——它知道"人体密度≈1000"、"鞋码推脚面积"这些常识，但这些知识此前从未被系统性地抽出来、结构化、可供概率推断调用。**MEIS 的工程核心贡献就是搭这个桥**。

与目前所有 agentic science 系统最大的区别：**MEIS 不是让 LLM 去回答问题，而是让 LLM 去喂养一个可推断的信念网络，然后由网络回答问题。** LLM 的角色从"裁判"退回到"证人+翻译"——这更符合它的真实能力边界，也避开了 LLM 幻觉在关键推理环节的污染。
