## 那篇论文在做什么

Wong, Grand, Lew, Andreas, Goodman, Tenenbaum 这一组（2023，arXiv:2306.12672）提出的框架叫 **Rational Meaning Construction**，核心操作是：

**自然语言 → LLM 翻译 → 概率程序（在 Church/Gen 这类语言里）→ 概率推断引擎运行 → 结果返回**

举个他们论文里的例子。给定一段描述："Alice 比 Bob 高，Bob 和 Charlie 一样高"，然后问"Alice 和 Charlie 谁高？"。LLM 不直接回答，而是把这段话翻译成一段概率程序——定义变量 Alice.height、Bob.height、Charlie.height，加上约束 Alice.height > Bob.height、Bob.height == Charlie.height——然后交给概率引擎做推断，引擎返回 P(Alice.height > Charlie.height) ≈ 1。

他们在四个领域验证了这个架构：概率推理、关系推理、物理场景理解、目标/规划推断。每个领域都用同一个模式：LLM 做**语义解析**，概率程序做**推断**。

关键论点是 LLM **不该**承担推理本身。LLM 擅长理解语言意思，但概率性、组合性、需要精确追踪的推理应该交给一个显式的、可检查的符号-概率混合引擎。

## 你描述的东西在 AI 研究里有个名字

**贝叶斯网络里**叫"证据传播"（evidence propagation）或"信念传播"（belief propagation）。Pearl 1988 年那本《Probabilistic Reasoning in Intelligent Systems》就是奠基作。给定一个 DAG，节点是变量（身高、体重、密度、鞋码、脚印深度），边是条件概率关系，观测某些节点后所有其他节点的后验自动更新。你描述的"知道鞋码和脚印深度就能断定体重关系"就是这个机制——每多观测一个节点，未观测节点的后验方差就缩小。

**因果推断里**是 Pearl 后来 do-calculus 那套，Judea Pearl《The Book of Why》有通俗版。你说的"密度差不多"这种先验其实是一个**结构性假设**——即 weight = density × volume 这个物理关系被编码进了图结构里，不是可观测数据。这种"把机制知识编码进图"是因果建模的核心。

**侦探推理**你直觉对——Peirce 把溯因推理（abduction）看作科学方法的第一步，福尔摩斯式推理在认知科学里被形式化为 **Inference to the Best Explanation**（Harman 1965, Lipton 2004）。现代版本就是贝叶斯的 MAP 估计加上模型比较，这和你说的"挑选最小嵌入的理论"数学上是同一件事。

## 真正在做这件事的工作

按你的具体愿望（跨域先验库 + 传递式信念收紧 + LLM 接口）来看，最接近的有几个：

**Probabilistic Graphical Models + LLM 的结合**是一个正在起的方向。2023-2024 有几篇工作做 LLM 提取知识构建贝叶斯网络，然后在图上做推断。比如"Causal Reasoning and Large Language Models"（Kıcıman et al. 2023）让 LLM 输出因果图结构。但这些工作的图都很局部，没有你说的那种"跨域链式传递"。

**Tenenbaum 组的另一条线**——不是 Rational Meaning Construction 那篇，而是他们十几年来做的**直觉物理引擎**和**直觉心理学引擎**（Battaglia, Hamrick, Ullman 等人的工作）——其实更接近你的直觉。他们假设人脑里有一个**可模拟的物理世界模型**，看到场景后在脑中运行模拟来预测。"积木塔会不会倒"、"这个物体多重"这类判断就是你说的跨域先验传递（形状→材质→重量→稳定性）。但这条线主要做感知任务，没有和语言深度结合。

**Gärdenfors 的概念空间**（Conceptual Spaces, 2000）是一个哲学-认知科学工作，主张概念用几何空间的凸区域表示，不同维度（颜色、大小、重量、密度）有几何结构，推理就是在这个空间里做几何操作。你的"最小修正嵌入"在这个框架里对应**最短几何距离的概念调整**。理论漂亮，但工程实现非常少。

**Josh Tenenbaum 的"game engine in the head"愿景**——他在很多演讲里提过，人脑用一个通用游戏引擎式的世界模型做推理，身高→体重→压强→脚印这种链条就是引擎里的物理规则在跑。他承认这个愿景还没实现，Rational Meaning Construction 是朝这个方向的一步。

## 为什么还没做成

有个工程-哲学双重瓶颈。

工程上，跨域先验库是个**巨型标注工程**。"密度相近"、"形状相似时体积正比于某尺度立方"、"压强=重量/接触面积"——这些先验要么手写（不 scalable），要么从 LLM 抽取（噪声大、不一致），要么从数据学（需要结构化数据）。三条路都没真正打通。

哲学上有个更深的问题：你说的"密度差不多"这个先验**强度**怎么定？两个身材相似的人密度差异 ±5%？±2%？这个数字从哪来？人在做"最小嵌入推理"时显然没在脑子里算具体数字，而是做了某种**定性**贝叶斯。定性概率推理（qualitative probabilistic networks, Wellman 1990）是个老领域但一直没大突破。

## 所以你的原想法其实更完整

回到你最初的表述——"LLM 提供世界已有的样子 + 世界模型负责嵌入缺失的逻辑 + 用这个视角评价"——**这个三段式架构是对的**，而且你加上"跨域先验传递"这个维度后，它比 Rational Meaning Construction 更接近 Tenenbaum 那一派十几年的终极愿景。

具体差异可以这样对照：Rational Meaning Construction 是**任务内**的概率推断，用 LLM 做前门翻译；你描述的是**跨任务累积**的概率推断，用 LLM 做知识注入 + 假设生成，用概率图做传递和评价。前者是后者的一个切片。

如果真要实现，你会发现最硬的骨头不是 LLM 那一端（LLM 已经够用了），也不是概率推断那一端（贝叶斯网络、因果图、MCMC 都是成熟技术），而是**中间那层先验库的结构和粒度**。这正是为什么 Cyc（三十多年手工知识库项目）失败了，ConceptNet 只能做浅层关联，而 LLM 虽然"知道"这些先验但提不出结构化形式。

这个中间层是当前的真缺口。谁先解决它，谁就握住了你描述的这类系统的钥匙。

## 你这个问题的真正数学结构

让我把你的身高-体重-脚印例子形式化一下：

- 变量节点：height、weight、density、volume、shoe_size、foot_area、pressure、footprint_depth
- 结构方程（编码领域先验）：weight = density × volume，volume ≈ f(height)（同体型人近似比例），pressure = weight / foot_area，footprint_depth = g(pressure, 土壤硬度)
- 不确定性先验：density 约为 1000 kg/m³ ± 5%（人体密度范围），volume 与 height 的关系有一定噪声

这**就是一个贝叶斯网络**。观测 height、shoe_size、footprint_depth 后，对 weight 做信念传播（belief propagation），后验方差就会按你直觉的方向收紧。Pearl 1988 年就把这套机制讲完了。每多一个观测对应的证据进入网络，所有未观测节点的后验自动更新——这正是你说的"从 50:50 逐步逼近 100%"。

所以对这个具体子问题，你需要的是 **Probabilistic Graphical Models + 结构方程**，不是范畴论也不是黎曼流形。

## 那范畴论和信息几何在哪里有用

它们在这个问题里各自对应一个**不同的**增值点，但都不是核心机制：

**信息几何有一个精确的应用**——Fisher information matrix 告诉你哪个观测对减少 weight 的不确定性贡献最大。在你的例子里，它能定量回答："如果我只能再问一个问题，问鞋码更有用还是问脚印深度更有用？" 这是 active inference / 最优实验设计的数学基础。Chentsov 定理保证了 Fisher 度量在这个问题上是唯一自然的选择。

**范畴论的应用不在单次推理上，而在跨域识别上**。当你遇到一个新问题——比如"电压 V、电阻 R、电流 I、功率 P"——范畴论能帮你识别：这个推理结构与 height-weight-pressure 的结构是**同构的**（都是 Markov 范畴中同一类合成态射）。识别出同构后，你可以把"身高推体重"这个子图学到的信念传播策略，直接迁移到"电压推功率"上。这是我们上次讨论的那件事：**结构性的特征提取**。

换句话说：范畴论+信息几何回答的不是"如何做一次跨域推理"，而是"如何识别不同跨域推理之间的共同抽象"和"如何度量在某个已形式化的假设空间里两点的距离"。

## 为什么这个分工正好印证了上次的结论

你看，这和我们上次收敛到的那个分层架构完全吻合：

- **表示层**——贝叶斯网络/概率编程（把 height、weight 这些写下来，把 density 先验写下来）
- **度量层**——信息几何（Fisher 信息告诉你哪个观测最值钱）
- **结构层**——范畴论（识别 height-weight-pressure 和 V-R-I-P 是同一类结构）
- **调度层**——LLM+递归循环（提出"试试看脚印能不能帮助推断"这类假设）

你这次的问题其实是**表示层和度量层的结合**，所以核心工具是贝叶斯网络 + Fisher 信息，范畴论在这里只是在未来做泛化时才派上用场。

## 真正的瓶颈还是那个瓶颈

但你会发现，就算有了这些数学工具，你还是没法自动解决"Alice 和 Charlie 谁体重重"这个问题。因为系统不知道要去调用"人体密度≈1000、±5%"这个先验。这个先验**不在任何数学框架里**——它是一条常识性世界知识，躺在 LLM 的权重里，也躺在人类的直觉里，但没躺在任何可直接计算的结构里。

这就回到了两次对话都指向的同一个缺口：**跨域先验库的表示和填充**。范畴论+信息几何再精致，没有先验就是空架子。侦探能断案不是因为他的概率运算能力强，而是因为他脑子里有一个巨大的"各种物品/人/行为之间关系先验"的网络——这个网络的**规模和填充方式**才是真正未解决的问题。

所以诚实的答案是：范畴论+信息几何能处理这个问题的"运算"层，但处理不了"知识注入"层。而后者才是让 LLM 真正变成"能做最小嵌入推理的智能体"的卡点。

- 自动微分直接支持 Fisher 信息矩阵计算（Phase 2 度量层需要）
- Python 生态和 LLM API 无缝对接
- 社区规模大，debug 成本低
- 如果后面要做 Phase 4 的范畴论，Julia 生态会更有优势，但那是 12 个月后的事

**2. 它没有持久信念网络。** 每个 Tug-of-War 示例是一次性的 Church 程序，跑完即弃。你 Phase 2 要的"跨域先验库"在这个 repo 里完全没有——这是你的原创贡献点。

POPPER 对 MEIS 的价值：它提供了"给定一个自然语言假设，如何系统性地 falsify"的完整管道。你可以在你的 Phase 3 最小扰动评价的基础上**加一个 POPPER 式的证伪循环**作为下游模块。

## 二、没有 benchmark 的那一块 —— 自建"定律动物园"

你 MEIS 计划里 Phase 5 的核心贡献之一就是这个——**因为现有 benchmark 都不测"结构同构识别"**。

HypoBench 不测、DiscoveryBench 不测、POPPER 不测。没有任何 benchmark 问："RC 电路衰减和放射性衰变是否属于同一个抽象定律类？"

所以这里你必须自建，我上次在 MEIS 计划 Phase 5 里画的那个表就是起点：

```python
# 最小可行版本
laws = {
    "exponential_decay": ["RC_circuit", "radioactive", "first_order_reaction", "forgetting_curve"],
    "damped_oscillation": ["LC_circuit", "pendulum", "predator_prey_linearized"],
    "saturation_growth": ["logistic", "michaelis_menten", "capacitor_charging"],
    "allocation": ["gravity_field", "electric_field", "nash_equilibrium"]
}
# 每条定律用 50-100 条时间序列表示，附带 ground truth 标签
```

这本身就是一篇 benchmark 论文的种子——"**IsomorphBench: Evaluating Cross-Domain Structure Recognition in Scientific Reasoning**"——因为这个空位是真实存在的。

## 四、一个常被忽略的实用建议

**你会发现这些 benchmark 彼此数据格式不兼容**。bnlearn 用 .bif，HypoBench 用 JSON，POPPER 用专用 Python API，DiscoveryBench 用自己的 schema。所以实际上你第一周除了 fork world-models repo 之外，还要做一件同样重要的事：

**写一层数据适配器**，把这些 benchmark 统一到你的内部表示（变量 + 关系 + 观测 + ground truth 四元组）。这个工作看起来不起眼，但会在后面六个月里反复给你节省时间。

## 五、补充一句老实话

POPPER 论文的第一作者 Kexin Huang 明确在论文里说：**"many real-world hypotheses are abstract, high-level statements that are difficult to validate directly"**——这个领域本身承认评测是个半开放问题。没有任何单一 benchmark 能完全评价 MEIS 这样一个"融贯度评分系统"。

所以你的 Phase 5 自建评测环境，不是补丁，是贡献——很可能是这篇博士论文三篇论文里**最容易被引用**的那一篇，因为它填的是基础设施空位（类比 Gym 之于 RL）。

## 最直接继承：Michael Y. Li & Stanford 线（最有实验和代码）

这条线是实际把 Wong 那篇的愿景**落地为可跑的实验系统**，Noah Goodman 是连接点（他同时是 Wong 论文和这条线的作者）。

### 1. Automated Statistical Model Discovery with Language Models (ICML 2024)
**Li, Fox, Goodman** | arXiv:2402.17879

这是最接近原论文精神的工程延伸。把 Wong 框架套进 Box's Loop 做**迭代式科学发现**——LLM 既当 modeler（提出概率程序）又当 critic（诊断模型缺陷），在真实数据集上循环优化。用 PyMC 而不是 Church，这对你迁移到现代技术栈是直接的范本。

### 2. BoxingGym (2025) ⭐ **对你最重要**
**Gandhi, Li, Goodyear, Bhatia, Goodman** | arXiv:2501.01540 | GitHub: https://github.com/kanishkg/boxing-gym

**这是你上一轮问"没 benchmark 怎么办"的最佳答案**。10 个用 pymc 实现的生成式模拟环境，专门评估 LLM 做**实验设计 + 模型发现**两件事。指标是 **Expected Information Gain (EIG)**——这正好是你 MEIS Phase 2 里"哪个观测最值钱"的数学化版本。

关键发现：**当前 LLM（含 GPT-4o）在这些任务上表现很差，而且给它加一个显式的统计模型也不会可靠地改善结果**——这对你的 MEIS 是个有趣的验证机会：如果你的跨域先验库 + 最小扰动度量能在 BoxingGym 上超越他们的 baseline，就是一篇有说服力的论文。

PyMC 生态 + 开源 + 可扩展 + 有 leaderboard——你的 Phase 1/2 评测应该优先打这个。

### 3. CriticAL (2025)
**Li, Vajipey, Goodman, Fox**

在 Box's Loop 基础上加了一个"LLM 生成统计假设检验"的批评者。把 LLM 当 verifier 嵌入假设检验框架。这对你 Phase 3 的最小扰动评价函数的"负面样例生成"是个好工具。

## 核心框架扩展：Ying + Zhi-Xuan + Wong 线（MIT 社会推理方向）

这条线是 Wong 论文作者自己在做的延伸，但主要朝"社会/心智推理"走，**实验做得更严谨但领域偏窄**。

### 4. LaBToM (2024-2025)
**Ying, Zhi-Xuan, Wong, Mansinghka, Tenenbaum** | arXiv:2408.12022 (TACL 2025)

"Language-augmented Bayesian Theory of Mind"——字面上就是把 Wong 框架应用于信念归因。**做了真实人类实验**：让受试者看 agent 在迷宫里找钥匙，对有关 agent 信念的句子打分；LaBToM 与人类判断相关性显著高于 GPT-4o、Gemini Pro 和消融模型。

这是 Wong 框架**首个有大规模人类基准对照的实现**。

### 5. LIRAS (CogSci/CoLM 2025)
**Ying, Truong, Collins, Zhang, Wei, ..., Wong, Tenenbaum** | arXiv:2506.16755

"Language-Informed Rational Agent Synthesis"——把 LaBToM 扩展到**多模态**（语言+视觉），用 VLM 解析场景到 PDDL 符号表示，贝叶斯反向规划做推断。在多个认知科学任务上超越消融和 SOTA。

### 6. AutoToM (2025)
**Ying, ..., Shu** — "Automated Bayesian Inverse Planning and Model Discovery for Open-ended Theory-of-Mind"

进一步做模型**自动发现**（不再手写 generative model，让系统自己建模）。这个朝向和你 MEIS Phase 3-4 是高度一致的。

## 最新的综合尝试：MSA 和 Language-and-Experience 线

### 7. Model Synthesis Architecture (MSA)
**Wong, Collins, ..., Brooke-Wilson**

Lionel Wong 自己的延续工作。提出"distributed + symbolic 混合表示"——LLM 做全局相关性检索和模型合成，概率程序做局部连贯的世界模型。这比原论文更接近你 MEIS 里"LLM 调度 + 信念网络 + 跨域先验"的三层架构。

### 8. Language and Experience (ICLR 2026)
arXiv:2509.00074

把 LLM + 概率程序 + 贝叶斯推断结合起来**学游戏规则**——一个 agent 看先前玩家的文字建议 + 自己玩 10 次，逐步精化一组"候选游戏规则程序"。**这是第一个在开放、交互式环境里跑通整个 Wong 框架闭环的工作**。

## Wong 自己的博士论文（2024）

**From Words to Worlds: Bridging Language and Thought** | MIT, 2024
https://dspace.mit.edu/bitstream/handle/1721.1/157326/wong-zyzzyva-phd-bcs-2024-thesis.pdf

Lionel Wong 把原 arXiv 论文扩展成博士论文，**里面有实验章节**（"Experiment overview / Methods / Results / Discussion"），在人类实验和 LLM 消融上都做了对照。你应该读这篇——它比 arXiv 版本完整得多，详细到"Concretely implementing Rational Meaning Construction models"这一章把工程细节都展开了。
